## 💡 프롬프트 잘 사용하는 방법 (조사 내용)

### 1단계: 목표 명확히 하기
>
> 프롬프트에 반드시 목적을 명확히 적어주세요.  
예시:  

- "CS 지식으로 밈을 만들고 싶어요."  
- "아래 체크리스트를 바탕으로 시간표를 만들어주세요."

### 2단계: 배경 정보 제공하기
>
> AI가 참고할 수 있도록 필요한 정보를 구체적으로 입력하세요.  
예시:  

- "아래는 제가 정리한 CS 내용입니다. 이 내용을 바탕으로 밈을 만들어주세요."  
- "아래는 저의 체크리스트입니다."

### 3단계: 원하는 결과물의 형식/스타일 제시하기
>
> 결과물이 어떤 형태이길 원하는지 구체적으로 요청하세요.  
예시:  

- "밈은 유머러스하게, 1~2문장으로 만들어주세요."  
- "시간표는 표 형태로, 각 항목별 예상 소요 시간을 포함해주세요."  
- "퀴즈는 객관식 3문제로 만들어주세요."

### 4단계: 추가 조건/제한사항 명시하기
>
> 특정 조건, 시간, 난이도 등 추가 요구사항이 있다면 명확히 적으세요.  
예시:  

- "현재 시간은 오전 10시입니다. 오전 시간대에 집중할 수 있도록 시간표를 짜주세요."  
- "퀴즈 난이도는 중급으로 해주세요."

### 5단계: 역할 부여하기 (필요시)
>
> AI에게 역할을 부여하면 더 자연스러운 답변을 받을 수 있습니다.  
예시:  

- "당신은 CS 전문가입니다. 아래 내용을 바탕으로 퀴즈를 만들어주세요."  
- "당신은 유머러스한 밈 제작자입니다."

### 6단계: 예시 답변 요청하기 (선택)
>
> 원하는 답변 예시를 함께 제시하면 더욱 원하는 결과를 얻을 수 있습니다.  
예시:  

- "예시) 'TCP와 UDP의 차이점: TCP는 택배, UDP는 쪽지!' 이런 식으로 만들어주세요."

#### `문제 답 제출`
일단 내가 이 부분에 대해서 확실하게 이해했다고 생각해서 다음과 같이 문제를 풀어서 제출하였다.
```
1. add를 하면 현재 README.md 파일 내용을 읽고 특정 형식으로 변환한다음, 해당 내용을 가지고 SHA해시값을 얻어. 이후 압축한 다음 blob객체로 만들어서 objects 폴더에 저장해. 그리고 스태이징 상태로 만들기 위해서 위 정보를 index파일에도 저장시켜. 저장되는 정보는 파일모드, 해시값, 파일이름 등이야.
2. root - README.md, root-src-App.js 트리 구조가 생성돼.각 트리 객체에는 해당 디렉토리에 저장되어있는 blob과 Tree 객체 정보가 저장돼.
3. 커밋을 하게되면, commit 객체에는 Root Tree 객체 정보다 들어가고 그밖에 메타 정보 (작성자, 커밋한 사람, 시간정보, 커밋메시지)가 들어가게돼. 그리고 이전 커밋과 연결할 수 있는 부모 커밋객체 정보도 저장돼.
4. HEAD에는 현재 어떤 브랜치를 가리키고 있는지 ref 정보가 저장돼. 그리고 해당 HEAD를 따라가면 브랜치 파일이 있는데 여기에는 현재 해당 브랜치가 어떤 커밋 해시값을 가리키고 있는지가 나와. 따라서 커밋시에는 branch 파일 내용이 커밋한 객체로 바뀌게 돼. 이를통해 branch를 구분할 수 있지.
5. 파일모드는 일반파일, 디렉토리, 실행파일, 심볼 파일 등이 있어. 의미는 이름 그대로야.
6. 동일한 내용의 파일을 add하면 동일한 해시값을 포인팅하게 돼. 왜냐하면 파일 내용이 동일하면 git은 새로운 파일을 만드는게 아니라, 기존에 있는 파일을 사용하거든. 이를 통해 git에서는 불필요하게 중복된 파일을 없앨 수 있어서 공간을 효율적으로 사용할 수 있지. 이를 가능하게 하는 핵심은 파일의 참조(이름)을 저장한다는 거지.
```
