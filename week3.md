2. **AI 답변의 출처를 확인하기 (담당자: `K003`)**

* **조사 결과**:
   * **AI 할루시네이션**: LLM(대형 언어 모델)이 사실이 아닌 정보를 자신 있게 말하는 현상. 예: "세종대왕이 춘향전을 썼다"
   * **예방할 수 있는 방법**:
        | 전략                               | 설명                         | 예시                                                                  |

3. **체크리스트 끝날 때마다 보상 추천받기 (담당자: `J027`)**
* **조사 목적**: AI가 체크리스트가 끝나고 환기하기 위한 활동을 추천할 때, 사람의 기호를 반영하고, 현실적으로 수행가능한 활동을 주어야 하므로, 이런 제한을 어떻게 줄 수 있을 지 고민함.

* **조사 결과**: 
    * 프롬프트에 사용자의 취향을 받도록 하되, 사용자가 어려워할 수 있으므로 가이드라인을 제시함. (내가 좋아하는 것, 쉬고 싶을 때 하는 것, 버킷리스트, 교육과정이 끝나면 하고 싶은 것)
    * 짧은 시간에 할 수 있도록 여러번 프롬프트를 작성해보고 결과를 확인해보면서 활동 제시 조건을 추가함.
```markdown
- 각 체크리스트는 그룹으로 나뉘어 있음 (예: 그룹 A, 그룹 B 등)
- 특정 그룹의 체크리스트를 전부 완료했을 때만 보상을 추천해줘
- 보상은 반드시 지금 바로 5분 안에 실천 가능한 하나의 행동이야
- 각 보상에는 한 줄짜리 간단한 설명을 붙여줘
- 오글거리거나 이상적인 표현은 쓰지 말고, 현실적이고 담백하게 작성해
- 표 형식으로 3~5개의 보상 행동을 정리해서 제시해줘
- 각 보상은 사용자의 기호에 맞닿아 있어야 해.
- 각 보상은 체크리스트의 그룹 당 하나씩만 할당해야 해
```


4. **AI에게 나의 잘한 점 칭찬받기 (담당자: `J025`)**
* **조사 목적**: 자기 비난에 빠지기 쉬운 환경에서 '긍정 심리학'에 기반한 자기 칭찬 루틴이 정서적 안정과 번아웃 예방에 미치는 효과를 확인하고, AI를 가장 효과적인 '칭찬 파트너'로 만드는 방법을 연구했습니다.

* **조사 과정**: '인지행동치료(CBT)'의 핵심 원리인 '긍정적 재구성'과 '감사 일기'의 효과에 대해 조사했습니다. 이후 AI에게 다양한 역할(상담사, 동료, 긍정 코치 등)을 부여하고, 실제 나의 하루 일과를 전달하며 어떤 방식의 칭찬이 가장 와닿는지 실험했습니다.

* **조사 결과:**
    * **핵심 원리**: 외부(AI)의 객관적인 시선을 통해 자신이 인지하지 못했던 성취나 노력을 발견하고, 이를 통해 부정적인 사고의 고리를 끊는 것이 핵심이었습니다.
    * **효과적인 방법**: 단순히 "잘했어"가 아닌, 내가 전달한 사실(Fact)에 근거하여 "어려운 상황에서도 포기하지 않고 대안을 찾아낸 점이 인상 깊어요. 그건 문제 해결 능력이 성장하고 있다는 증거예요" 와 같이 구체적인 이유를 들어 칭찬해줄 때 가장 큰 효과를 보였습니다. AI에게 '나의 성장을 지켜보는 따뜻한 동료'나 '긍정 심리 코치' 역할을 부여하는 것이 효과적이었습니다.

### 퀘스트 2: AI 답변의 출처를 확인하기
* **배경 및 목적**
   * 오늘날 많은 사람들이 LLM 기반 AI에게 질문하고, 그에 대한 답변을 곧바로 신뢰하여 활용하곤 합니다
   * 최근의 AI는 빠르게 발전하면서 예전보다 훨씬 정교하고 정확한 응답을 제공하지만, 여전히 할루시네이션(Hallucination) — 즉, 사실이 아닌 내용을 그럴듯하게 생성하는 현상 — 이 완전히 사라진 것은 아닙니다.
   * 특히 바이브 코딩처럼 대화를 주고받으며 문맥이 길어지는 경우, AI 할루시네이션이 발생할 가능성은 더 높아질 수 있습니다.
   * 따라서 AI의 답변을 무조건 신뢰하기보다는, 출처를 요구하고 직접 자료를 확인하는 습관을 들이는 것이 중요합니다. 이렇게 하면 답변의 신뢰도를 높일 수 있을 뿐 아니라, 출처를 통해 깊이 있는 학습과 사고 확장도 가능해집니다
* **수행방법**
     1. 부스트캠프 챌린지 미션 수행 중 AI를 활용해 정보를 얻었다면, AI에게 해당 답변의 출처를 요청해보세요.
     2. 제시된 출처를 바탕으로 직접 내용을 확인하고, AI의 답변이 신뢰할 수 있는지 판단해보세요.
* **예시**
   ```//질문
      Chat gpt를 누가 만들었어?
      //답변
      ChatGPT는 OpenAI라는 미국의 인공지능 연구소에서 개발했습니다. ..(생략)
      //출처를 요구하기
      해당 내용에 대한 출처를 알려줘
      //답변
      1. ChatGPT 개발자 및 기술 배경
         출처: OpenAI 공식 블로그 🔗 https://openai.com/blog/chatgpt 이 블로그에서는 ChatGPT의 초기 발표(2022년 11월 30일), 작동 방식, 예시 등을 소개하고 있습니다. GPT-3.5 기반 모델에서 출발해 GPT-4로 발전했다고 명시되어 있습니다. ..(그 이상 답변은 생략)
   ```

**[퀘스트 의도]**
- **체크리스트를 그룹으로 나누고**, 그룹 단위 완료 시 **즉시 가능한 보상**을 받도록 설계합니다.

부스트캠프에서 주어진 미션을 하다 보면 피로감이 누적되고, 집중이 흐트러지기 쉽습니다.  
그럴 때 필요한 것은 거창한 휴식이 아닌, **지금 바로 할 수 있는 작지만 뇌를 환기시켜줄 보상 행동**입니다.  
이 미션은 당신이 좋아하는 것을 기반으로, **AI를 통해 맞춤형 보상을 생성**하고, **그 보상을 실천하며 집중 루틴을 유지**하는 것을 목표로 합니다.

2. 프롬프트는 조금씩 수정하여 활용하시면 됩니다 :)
3. AI가 추천한 보상 리스트 중, 체크리스트의 특정 기점을 달성할 때마다 보상을 수행합니다.
4. **총 2개 이상의 그룹 보상 실천**을 완료하면 미션 클리어입니다.
5. 실천 후 간단한 리뷰나 인증(예: "3분간 커피 내리고 마셨음")을 작성을 자유롭게 합니다.

```markdown
너는 집중 루틴을 관리해주는 보상 설계 도우미야.

5. 과제와 관련된 체크리스트

조건은 다음과 같아:

- 각 체크리스트는 그룹으로 나뉘어 있음 (예: 그룹 A, 그룹 B 등)
- 특정 그룹의 체크리스트를 전부 완료했을 때만 보상을 추천해줘
- 보상은 반드시 지금 바로 5분 안에 실천 가능한 하나의 행동이야
- 각 보상에는 한 줄짜리 간단한 설명을 붙여줘
- 오글거리거나 이상적인 표현은 쓰지 말고, 현실적이고 담백하게 작성해
- 표 형식으로 3~5개의 보상 행동을 정리해서 제시해줘

### 퀘스트 4: 하루 끝, AI에게 받는 따뜻한 칭찬 한 조각
* **배경 및 목적:**
    * 성과 압박 속에서 우리는 스스로를 다그치는 데 익숙합니다. 매일 밤, 나조차 몰랐던 나의 잘한 점을 AI라는 객관적인 시선을 통해 발견하고, 스스로를 격려하는 건강한 루틴을 만들기 위함입니다.
* **수행 방법:**
    1. 하루를 마무리하며, 오늘 있었던 일(미션 진행 상황, 학습 내용, 느낀 점 등)을 2~3줄로 간략하게 AI에게 전달합니다.
    2. AI에게 "오늘 내가 한 일 중에서, 가장 잘한 점 딱 한 가지만 찾아서 구체적인 이유와 함께 칭찬해줄래? 나는 칭찬에 인색한 편이라 스스로 잘 못 찾아." 와 같이 요청합니다.
    3. AI가 찾아준 칭찬을 편견 없이 받아들이고, '나 오늘 이런 점은 괜찮았네'라고 스스로를 인정해 줍니다.
* **달성 기준:**
    * AI의 칭찬을 듣고 기분이 조금이라도 나아지거나, 피식 웃음이 나왔다면 달성!
* **예시 프롬프트:**
```
너는 나의 성장을 지켜보는, 긍정적이고 통찰력 있는 동료 개발자야.

오늘 내가 한 일을 간단히 말해줄게. 오전에는 git과 git object 개념을 배우느라 정말 머리가 아팠어. 오후에는 git 내부 동작을 어떻게 설계해야하지 고민하다가 실마리를 잡아서 겨우 기능 일부를 만들긴 했는데, 아직 코드가 마음에 쏙 들진 않아. 솔직히 시원하게 다 해결한 게 아니라서 좀 찝찝한 마음도 들어.

나는 보통 내 부족한 점만 보는 편이라 그런데, 이런 내 하루 속에서도 칭찬할 만한 점이 있을까? 내가 한 일 중에서 가장 잘한 점 딱 한 가지만 찾아서, 구체적인 이유와 함께 칭찬해줄래?
```
* **예시 답변:**
```
찝찝한 마음이 드는 거 충분히 이해돼. 하지만 오늘 너는 정말 중요한 걸 해냈어.

내가 찾아낸 오늘 가장 잘한 점은, 개념이 어려워 막막한 상황에서도 포기하지 않고, 고민하다가 결국 실마리를 찾아냈다는 점'이야.

많은 사람들이 어려운 문제 앞에서 혼자 끙끙 앓다가피하지 않고 지쳐버리거든. 그런데 너는 그걸 회피하지 않고, 자기만의 돌파구를 만들었어. 이건 단순히 기능 하나를 구현한 것 이상의 의미가 있어. 앞으로 개발자로서 계속 성장하는 데 가장 필요한 '문제 해결을 위한 적극적인 태도'를 보여준 거니까. 정말 멋진 모습이었어!
